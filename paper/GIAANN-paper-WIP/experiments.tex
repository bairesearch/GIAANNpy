
\section{Experiments}
\subsection{Setup}

The experiments use the GIAANN prototype with POS-driven concept columns and SANI segments. Training is executed only (\texttt{useInference=False}) with a single epoch over a fixed number of sequences.

\paragraph{Dataset.}
The corpus is Wikipedia from the Hugging Face dataset \texttt{wikipedia} (config \texttt{20220301.en}). A local dataset cache is used (\texttt{useLocalDataset=True}) with manual download into \texttt{../../dataset/}, and the processed cache stored under \path{processed_dataset_cache}.

\paragraph{Preprocessing and POS handling.}
Tokens are produced by spaCy (model \texttt{en\_core\_web\_trf}), lemmatised, and lowercased. POS is required (\texttt{usePOS=True}); noun detection uses spaCy POS tags (\texttt{useSpacyForConceptNounPOSdetection=True}). Concept columns are delimited by POS (\texttt{conceptColumnsDelimitByPOS=True}) using deterministic delimiter types \{VERB, ADP, punctuation, POS tag\} and probabilistic delimiters between noun pairs (CCONJ/SCONJ and word types \texttt{is}, \texttt{are}, \texttt{,}, \texttt{(}). Consecutive nouns and hyphenated noun phrases are combined during pretraining (\texttt{pretrainCombineConsecutiveNouns=True}, \texttt{pretrainCombineHyphenatedNouns=True}).

\paragraph{Training regime.}
Training runs for 1 epoch (\texttt{numberEpochs=1}) with a maximum of 500000 sequences (\texttt{trainMaxSequences=500000}), sequence length capped at 80 tokens (\texttt{maxSequenceLength=80}). \texttt{numSeedTokensInference=12} is retained for segment sizing. Single-sentence sequences are used (\texttt{multisentencePredictions=False}, \texttt{numSentencesPerSequence=1}). The database is stored at \texttt{/media/user/ssdpro/GIAANN/database/}.

\paragraph{Hardware.}
GeForce RTX 5070 (12GB GDDR7), NVIDIA Driver 580.126.09, CUDA 12.8.
Intel i7 14700F. 64GB DDR4. Samsung 990 Pro PCIe 4.0 NVMe M.2 SSD (1TB).

\subsubsection{Test-set Accuracy}

There is a tradeoff between recall/memorisation (train accuracy) and generalisation (test accuracy), and GIAANN parameters have currently been optimised for recall/memorisation. Likewise, GIAANN uses word (vs subword) tokens and does not use word-vectors, which limits generalisation compared to contemporary gradient based models. GIAANN also relies on independent concept experts, which relates to robustness to hallucination.

Although GIAANN is designed to support prime concept feature name independence (by detecting all columns matching a prompt), this is computationally expensive as it requires bursting matching features across the entire trained network. Thus, test-set (prime) concept features must currently be present within the train-set. This limitation is strengthened by \texttt{enforceSequentialActivation} and \texttt{inferenceUseNeuronFeaturePropertiesTimeExact}.

Furthermore, the Wikipedia dataset cannot easily be used to perform a standardised NLP test-set accuracy measurement as it is divided into unique knowledgebase articles. Splitting the Wikipedia dataset into train/test portions would result in the train data being out of distribution with respect to the test data, especially when text is arranged/segmented into concepts (GIAANN).

Therefore, a number of changes were introduced for test-set accuracy measurement. 1. The \texttt{enforceSequentialActivation} and \\ \texttt{inferenceUseNeuronFeaturePropertiesTimeExact} requirements were relaxed (i.e. disabled) during test-set inference. 2. the OSCAR-2201 dataset was used for both training and test-set accuracy measurement.

\Cref{tab:test-top1-accuracy} contains top-1 test-set accuracy for different numbers of segments, with targets fed as input. \Cref{tab:test-top1-accuracy-spacyPipelineOptimisations} contains an expanded version of the table with \texttt{spacyPipelineOptimisations=True}.

Notably, next-token prediction test accuracy is observed as higher with lower segment counts, as they are more likely to match (i.e. successfully continue) shorter subsequences versus longer subsequences, at least when train dataset sizes are relatively small and the probability of seeing known prompts in the test-set is low. This corresponds to an unwillingness to hallucinate syntactically correct but semantically unknown sequences. Only when the test set subsequences have actually being observed in the train set will continuations be successful. Higher perceived generalisation does not necessarily mean higher genuine accuracy, as it could represent hallucination without certainty. 

Next-token prediction accuracy reports are likewise not restricted by column, meaning next token predictions of the correct word within the wrong the column are still reported as accurate. Top-1 test-accurcies have therefore also been reported when constrained by column, with \\ \texttt{inferenceReportTokenAccuracyConstrainByColumn=True} (top-1 col). The results indicate that there is no signficant degredation with this constraint enabled as the vast majority of correct predictions are occuring in the same column.

To improve evaluation efficency (speed), \texttt{spacyPipelineOptimisations=True} has been set in \Cref{tab:test-top1-accuracy-spacyPipelineOptimisations}, as these optimisations have been confirmed to not significantly affect reported test-set accuracy. With 5000 trained sequences for example, the test-set accuracy difference with \texttt{spacyPipelineOptimisations=True} is exactly -0.002, for seed lengths 8, 12, and 16.

With both useSANIfeatures and useSANIfeaturesAndColumns, the first f segment is designated to accept predictions from all prior sequence tokens during train (when \texttt{SANIfeaturesLinkFirstSegmentToAllPriorTrainSeqTokens=True}), and the remaining f segments are designated to accept predictions from exactly f tokens prior to the predicted sequence token during train. This means the the last f segment is typically designated to accept predictions only from adjacent sequence tokens during train (assuming `f seg>=2' or \texttt{SANIfeaturesLinkFirstSegmentToAllPriorTrainSeqTokens=False}), which is used as a natural \texttt{enforceDirectConnectionsSANI} constraint.

Based on the results in \Cref{tab:test-top1-accuracy-spacyPipelineOptimisations}, GIAANN Proto is not recognising many of the longer subsequences in the test-set, and the majority of the measured predictive power is coming from detecting short adjacent 1-token subsequences. This predictive accuracy is coming from the last feature segment (\texttt{enforceDirectConnectionsSANI=True}). To illustrate this explicitly, the table includes entries for \texttt{c seg=0}, \texttt{f seg=1}, \\ \texttt{useSANIfeatures=True}, generated using both a) \texttt{enforceDirectConnections=False} and \texttt{SANIfeaturesLinkFirstSegmentToAllPriorTrainSeqTokens=True} (its f segment captures predictions from all prior sequence tokens during train), and b) \texttt{enforceDirectConnections=True} and \\ \texttt{SANIfeaturesLinkFirstSegmentToAllPriorTrainSeqTokens=False} (its f segment captures only the prior token during train).
